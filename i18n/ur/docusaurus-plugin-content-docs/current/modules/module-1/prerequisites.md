# ضروریات: اپنے ROS 2 ماحول کو سیٹ اپ کرنا

## جائزہ

ROS 2 اور ہیومنوائڈ روبوٹکس کی دلچسپ دنیا میں گہرائی سے جانے سے پہلے، ایک مضبوط اور کارآمد ترقی کا ماحول سیٹ کرنا ضروری ہے۔ یہ دستاویز ROS 2 Foxy یا Humble کو اوبنٹو سسٹم پر انسٹال کرنے، ایک ROS 2 ورک سپیس بنانے، اور `colcon` بلڈ سسٹم کو کنفیگر کرنے کے لیے تفصیلی ہدایات فراہم کرتا ہے۔ ایک درست طریقے سے کنفیگر کردہ ماحول ماڈیول 1 کے دوران ایک ہموار سیکھنے کا تجربہ کو یقینی بنائے گا۔

---

## اوبنٹو پر ROS 2 Foxy/Humble انسٹالیشن

یہ گائیڈ ROS 2 Foxy Fitzroy (برائے اوبنٹو 20.04) یا Humble Hawksbill (برائے اوبنٹو 22.04) کی انسٹالیشن کو کور کرتا ہے، دونوں ہی لمبی مدت کی حمایت (LTS) ریلیز ہیں۔ ہم اعلی مطابقت کے لیے متعلقہ اوبنٹو ورژن کی ایک تازہ انسٹالیشن کا استعمال کرنے کی سخت تجویز کرتے ہیں۔

### 1. لوکل سیٹ کریں

یقینی بنائیں کہ آپ کے سسٹم کی لوکل ترتیبات UTF-8 کی حمایت کرتی ہیں، جو ROS 2 کے لیے ضروری ہے۔

```bash
locale  # UTF-8 کی حمایت کے لیے موجودہ لوکل ترتیبات چیک کریں
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8 # موجودہ سیشن کے لیے لاگو کریں
```

### 2. ROS 2 ریپوزیٹری شامل کریں

اپنے سسٹم کے `apt` ذرائع میں ROS 2 GPG کی اور ریپوزیٹری شامل کریں۔

```bash
sudo apt update && sudo apt install curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(source /etc/os-release && echo UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

### 3. ROS 2 پیکیجز انسٹال کریں

اپنے پیکیج کی فہرست کو اپ ڈیٹ کریں اور ROS 2 کی مکمل ڈیسک ٹاپ انسٹالیشن انسٹال کریں۔

#### ROS 2 Foxy کے لیے (اوبنٹو 20.04 - Focal Fossa)
```bash
sudo apt update
sudo apt install ros-foxy-desktop
```

#### ROS 2 Humble کے لیے (اوبنٹو 22.04 - Jammy Jellyfish)
```bash
sudo apt update
sudo apt install ros-humble-desktop
```

### 4. ترقی کے اوزار انسٹال کریں

ROS 2 ترقی کے لیے ضروری اضافی Python اوزار انسٹال کریں، بشمول `colcon` اور آرگومنٹ کمپلیشن۔

#### ROS 2 Foxy کے لیے
```bash
sudo apt install python3-rosdep2 python3-colcon-common-extensions python3-argcomplete
```

#### ROS 2 Humble کے لیے
```bash
sudo apt install python3-rosdep python3-colcon-common-extensions python3-argcomplete
```

### 5. ماحول کا سیٹ اپ

ROS 2 کمانڈز کو اپنے شیل میں دستیاب بنانے کے لیے ROS 2 سیٹ اپ اسکرپٹ کو سورس کریں۔ مستقل رسائی کے لیے، اپنے شیل کی کنفیگریشن فائل (مثلاً `~/.bashrc`) میں سورس کمانڈ شامل کریں۔

```bash
# Foxy کے لیے (اپنے ٹرمنل میں یہ چلائیں):
source /opt/ros/foxy/setup.bash

# Humble کے لیے (اپنے ٹرمنل میں یہ چلائیں):
source /opt/ros/humble/setup.bash

# اسے مستقل بنانے کے لیے، ~/.bashrc میں شامل کریں (Humble کی مثال):
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc # تبدیلیاں فوراً لاگو کریں
```

اپنی انسٹالیشن کی تصدیق کے لیے ایک نیا ٹرمنل کھولیں اور ٹائپ کریں:
```bash
ros2 --version
```
آپ کو انسٹال کردہ ROS 2 تقسیم اور ورژن نمبر نظر آنا چاہیے۔

## ROS 2 ورک سپیس سیٹ اپ اور کنفیگریشن

ROS 2 ورک سپیس ایک منظم ڈائریکٹری ہے جہاں آپ اپنے کسٹم ROS 2 پیکیجز کو منظم کرتے ہیں، جس سے ان کو تعمیر کیا جا سکے اور اپنی اہم ROS 2 انسٹالیشن کے ساتھ ضم کیا جا سکے۔

### 1. ایک ورک سپیس ڈائریکٹری بنائیں

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws
```
یہ آپ کی ہوم ڈائریکٹری میں ایک ڈائریکٹری `ros2_ws` بناتا ہے، جس میں ایک `src` سب ڈائریکٹری ہے جہاں آپ کے ROS 2 پیکیجز واقع ہوں گے۔

### 2. `colcon` کے ساتھ اپنا ورک سپیس تعمیر کریں

`colcon` ROS 2 کا بنیادی بلڈ ٹول ہے۔ یہ مؤثر طریقے سے سورس کوڈ کمپائل کرتا ہے، ضروری فائلز تیار کرتا ہے، اور آپ کے پیکیجز کے لیے انحصار کو منظم کرتا ہے۔

```bash
cd ~/ros2_ws # یقینی بنائیں کہ آپ اپنے ورک سپیس کے روٹ میں ہیں
colcon build
```
جب آپ کسی نئے ورک سپیس میں پہلی بار `colcon build` چلاتے ہیں، تو یہ تمام دریافت شدہ پیکیجز کو کمپائل کرے گا۔ بعد کی تعمیرات صرف تبدیل شدہ پیکیجز کو دوبارہ کمپائل کریں گی، جو ترقی کو تیز کرے گا۔

### 3. اپنا ورک سپیس اوورلے سورس کریں

تعمیر کے بعد، آپ کو اپنے ورک سپیس کو اپنی ROS 2 انسٹالیشن کے اوپر "اوورلے" کرنا چاہیے۔ یہ آپ کے کسٹم پیکیجز اور ایگزیکوٹیبل کو ROS 2 کے ذریعے دریافت کے قابل بناتا ہے۔

```bash
source install/setup.bash
```
**اہم**: یہ کمانڈ اس وقت چلائی جانی چاہیے *جب آپ نے اپنی اہم ROS 2 انسٹالیشن کو سورس کر دیا ہو*۔ مستقل سیٹ اپ کے لیے، اسے اپنی `~/.bashrc` فائل میں شامل کریں، یقینی بنائیں کہ یہ `/opt/ros/<distro>/setup.bash` کو سورس کرنے والی لائن *کے بعد* آئے۔

```bash
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc # تبدیلیاں فوراً لاگو کریں
```

## `colcon` بلڈ عمل اور ٹربل شوٹنگ

### `colcon build` کمانڈ

*   **`colcon build`**: `src` ڈائریکٹری میں تمام پیکیجز کو تعمیر کرتا ہے۔
*   **`colcon build --packages-select <package_name>`**: صرف ایک مخصوص پیکیج اور اس کے انحصار کو تعمیر کرتا ہے۔ تیز تکرار کے لیے مفید ہے۔
*   **`colcon build --symlink-install`**: انسٹالیشن کے دوران علامتی لنکس تیار کرتا ہے، جو Python پیکیجز کے لیے بہت مددگار ہے۔ Python فائلز میں تبدیلیوں کو دوبارہ تعمیر کی ضرورت نہیں ہوتی۔
*   **`colcon build --event-handlers console_direct+`**: تعمیر کے عمل کے دوران مزید تفصیلی آؤٹ پٹ فراہم کرتا ہے، ڈیبگنگ کے لیے مفید ہے۔

### عام ٹربل شوٹنگ کی تجاویز

*   **"پیکیج نہیں ملا"**: یقینی بنائیں کہ آپ کا پیکیج آپ کے ورک سپیس کی `src` ڈائریکٹری میں ہے، اور آپ نے `colcon build` اور `source install/setup.bash` چلایا ہے۔
*   **انحصار غائب ہیں**: اگر `colcon` غائب انحصار کی رپورٹ دیتا ہے، تو `rosdep install --from-paths src --ignore-src -r -y` چلائیں۔
*   **تعمیر کی خامیاں**: ہمیشہ اپنے ورک سپیس میں تفصیلی خامی کے پیغامات کے لیے `log` ڈائریکٹری چیک کریں۔ اپنے پیکیج کے لیے `cmake`، `make`، یا `python` سے متعلق آؤٹ پٹ تلاش کریں۔
*   **ماحول سورس نہیں کیا گیا**: اگر `ros2` کمانڈز کو تسلیم نہیں کیا جاتا، یا آپ کے نوڈس نہیں مل رہے ہیں، تو یقینی بنائیں کہ آپ کا `~/.bashrc` درست طریقے سے کنفیگر کیا گیا ہے اور ہر نئے ٹرمنل میں سورس کیا گیا ہے۔
*   **صاف تعمیر**: کبھی کبھی، پرانے تعمیر کے آرٹیفیکٹس مسائل کا سبب بن سکتے ہیں۔ ایک صاف تعمیر اسے حل کر سکتی ہے:
    ```bash
    rm -rf install build log
    colcon build
    ```

## کلیدی نکات

*   ترقی کے لیے مناسب ROS 2 انسٹالیشن اور ماحول کا سیٹ اپ ضروری ضروریات ہیں۔
*   ROS 2 ورک سپیس کسٹم پیکیجز ترقی کے لیے ایک منظم سٹرکچر فراہم کرتا ہے۔
*   `colcon` ROS 2 کے لیے معیاری بلڈ ٹول ہے، جو کمپائلیشن اور انحصار کا انتظام کرتا ہے۔
*   ہمیشہ اپنا ماحول (ROS 2 بیس اور ورک سپیس اوورلے) درست طریقے سے سورس کریں۔
*   `colcon` نشاندہ اور ڈیبگ تعمیرات کے لیے مختلف اختیارات پیش کرتا ہے۔

## مشقیں

1.  **دہرائیں انسٹالیشن**: اگر ممکن ہو، تو کسی دوسرے مشین پر یا ورچوئل مشین میں (مثلاً VirtualBox کے ساتھ اوبنٹو) ROS 2 انسٹال کرنے کی کوشش کریں تاکہ عمل کو مضبوط کیا جا سکے۔
2.  **سیٹ اپ کی تصدیق کریں**: ایک نیا ٹرمنل کھولیں۔ `printenv | grep ROS` چلائیں۔ آپ کون سے ماحولی متغیرات دیکھتے ہیں، اور وہ آپ کے سیٹ اپ کے بارے میں کیا ظاہر کرتے ہیں؟
3.  **ایک جعلی پیکیج بنائیں اور تعمیر کریں**: اپنے ورک سپیس کی `src` ڈائریکٹری میں `my_dummy_package` نام کا ایک خالی ROS 2 Python پیکیج بنائیں۔ `colcon build --packages-select my_dummy_package` کا استعمال کرتے ہوئے صرف اس پیکیج کو تعمیر کریں۔ آپ اپنے ورک سپیس کی ڈائریکٹریز میں کون سی تبدیلیاں دیکھتے ہیں؟
4.  **غائب سورس کی شبیہہ بنائیں**: ایک نیا ٹرمنل کھولیں *بغیر* اپنے `~/.bashrc` یا کسی ROS 2 سیٹ اپ فائلز کو سورس کیے*۔ `ros2 run demo_nodes_cpp talker` چلانے کی کوشش کریں۔ آپ کو کیا خامی ملتی ہے، اور کیوں؟ پھر، دستی طور پر ROS 2 بیس سیٹ اپ (`/opt/ros/<distro>/setup.bash`) کو سورس کریں اور دوبارہ کوشش کریں۔
5.  **صاف تعمیر کی مشق**: اپنے ورک سپیس کی ایک صاف تعمیر کریں (`rm -rf install build log && colcon build`) کی مشق کریں۔ وضاحت کریں کہ یہ کب ضروری ہو سکتا ہے۔
