# موڈیول 1: ROS 2 فاؤنڈیشن

## جائزہ
موڈیول 1: ROS 2 فاؤنڈیشن میں خوش آمدید۔ یہ موڈیول روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک ROS 2، مڈل ویئر فریم ورک کا جامع تعارف فراہم کرتا ہے جو جدید روبوٹک ایپلی کیشنز کو طاقت دیتا ہے۔ آپ بنیادی تصورات، آرکیٹیکچر، اور پروگرامنگ پیٹرنز سیکھیں گے جو جٹیل روبوٹک سسٹم تیار کرنے کے لیے بنیاد کا کام کرتے ہیں۔ یہ موڈیول AI-ہیومنوائڈ روبوٹکس کے منصوبے کے تمام بعد کے موڈیولز کے لیے ضروری علم کو قائم کرتا ہے۔

ROS 2 روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک ہے جو ہارڈ ویئر ابھیلیشن، ڈیوائس ڈرائیورز، لائبریریز، ویژولائزرز، میسج پاسنگ، پیکج مینجمنٹ، اور مزید فراہم کرتا ہے۔ یہ جٹیل، تقسیم شدہ روبوٹک سسٹم کی ترقی کو سپورٹ کرنے کے لیے ڈیزائن کیا گیا ہے جس میں ریل ٹائم کارکردگی اور زیادہ قابل اعتمادی کی ضرورت ہوتی ہے۔

## سیکھنے کے اہداف
- ROS 2 کے بنیادی تصورات اور آرکیٹیکچر کو سمجھنا
- کور کمیونیکیشن پیٹرنز کا ماسٹر: ٹاپکس، سروسز، اور ایکشنز
- rclpy کے ساتھ Python پروگرامنگ میں مہارت حاصل کرنا
- URDF (متحدہ روبوٹ تفصیل فارمیٹ) ماڈلز بنانا اور کام کرنا
- متعدد مربوط نوڈس کے ساتھ مکمل ROS 2 سسٹم نافذ کرنا
- ROS 2 ترقی اور ڈیبگنگ کے لیے بہترین طریقے لاگو کرنا

## کلیدی تصورات
### ROS 2 آرکیٹیکچر
ROS 2 ایک تقسیم شدہ آرکیٹیکچر پر مبنی ہے جہاں کمپیوٹیشن متعدد عملوں اور ممکنہ طور پر متعدد مشینوں میں تقسیم ہوتا ہے۔ سسٹم DDS (ڈیٹا ڈسٹری بیوشن سروس) کو بنیادی کمیونیکیشن مڈل ویئر کے طور پر استعمال کرتا ہے، جو قابل اعتماد میسج ڈیلیوری اور دریافت کے میکنزم فراہم کرتا ہے۔

### نوڈس، ٹاپکس، سروسز، اور ایکشنز
ROS 2 کے بنیادی عمارتی بلاکس میں نوڈس (عمل جو کمپیوٹیشن انجام دیتے ہیں)، ٹاپکس (نامزد بسز جن کے ذریعے نوڈس میسجز تبدیل کرتے ہیں)، سروسز (درخواست/جواب کمیونیکیشن)، اور ایکشنز (Feedback کے ساتھ ہدف پر مبنی کمیونیکیشن) شامل ہیں۔

### کوالٹی آف سروس (QoS) سیٹنگز
ROS 2 میں QoS سیٹنگز کمیونیکیشن کے رویے کو ٹیون کرنے کی اجازت دیتی ہیں بشمول قابل اعتمادی، م durability، اور ہسٹری پالیسیز۔ یہ سیٹنگز ریل ٹائم اور سیفٹی کریٹیکل روبوٹک ایپلی کیشنز کے لیے اہم ہیں۔

### پیکج اور لانچ سسٹم
ROS 2 پیکجز کوڈ اور وسائل کو منظم کرتے ہیں، جبکہ لانچ فائلیں متعدد نوڈس کو مخصوص کنفیگریشن کے ساتھ ایک ساتھ شروع کرنے کے میکنزم فراہم کرتی ہیں۔

## تکنیکی گہرائی
ROS 2 اصل ROS فریم ورک کا ایک مکمل دوبارہ ڈیزائن ہے جو ریل ٹائم کارکردگی، سیکیورٹی، اور سکیل ایبلٹی کی ضروریات کو پورا کرنے کے لیے ہے۔ یہ آرکیٹیکچر DDS (ڈیٹا ڈسٹری بیوشن سروس) مڈل ویئر کے گرد تعمیر کیا گیا ہے، جو ریل ٹائم، سکیل ایبل، اور فالٹ ٹالرینٹ ڈیٹا ایکسچینج کے لیے ایک معیاری انٹرفیس فراہم کرتا ہے۔

### کور آرکیٹیکچر کمپوننٹس
ROS 2 آرکیٹیکچر میں کئی لیئرز شامل ہیں:
- **ایپلی کیشن لیئر**: صارف لکھے گئے نوڈس اور ایپلی کیشنز
- **کلائنٹ لائبریری لیئر**: rclpy (Python) اور rclcpp (C++)
- **ROS کلائنٹ لائبریری (rcl)**: مشترکہ C-بیسڈ انٹرفیس
- **DDS ابھیلیشن لیئر**: مختلف DDS ایمپلیمنٹیشنز کو ایبسٹریکٹ کرتا ہے
- **DDS ایمپلیمنٹیشن**: اصل DDS وینڈر ایمپلیمنٹیشنز (Fast DDS، Cyclone DDS، وغیرہ)

### تفصیل میں کمیونیکیشن پیٹرنز
**ٹاپکس** ایک پبلش-سبسکرائب پیٹرن نافذ کرتے ہیں جہاں پبلشر ٹاپکس پر میسجز بھیجتے ہیں بغیر سبسکرائبرز کے علم کے۔ سبسکرائبرز ٹاپکس سے میسجز وصول کرتے ہیں بغیر پبلشرز کے علم کے۔ یہ کم کپلنگ لچکیل سسٹم کمپوزیشن کو فعال بناتا ہے۔

**سروسز** ایک درخواست-جواب پیٹرن نافذ کرتے ہیں جہاں ایک کلائنٹ ایک درخواست بھیجتا ہے اور سرور سے جواب کا انتظار کرتا ہے۔ یہ ہم وقت کمیونیکیشن ایسے کاموں کے لیے بہترین ہے جن کو فوری نتائج کی ضرورت ہوتی ہے۔

**ایکشنز** ہدف پر مبنی پیٹرن کو نافذ کرتے ہیں جس میں Feedback اور نتیجہ رپورٹنگ شامل ہے۔ یہ طویل چلنے والے کاموں کے لیے مناسب ہے جہاں پیش رفت کی نگرانی ضروری ہے۔

### کوالٹی آف سروس پالیسیز
ROS 2 QoS پالیسیز میں شامل ہیں:
- **قابل اعتمادی**: قابل اعتماد بمقابلہ بہترین کوشش ڈیلیوری
- **م durability**: ولیٹائل بمقابلہ ٹرانزینٹ-مقامی ہسٹری
- **ہسٹری**: کیپ-آل بمقابلہ کیپ-لاسٹ پالیسیز
- **ڈیڈ لائن**: ڈیٹا ڈیلیوری کے لیے وقت کی پابندیاں
- **لیولی نیس**: نوڈ دستیابی کی نگرانی

## کوڈ مثالیں
### بنیادی ROS 2 نوڈ (Python)
```python
import rclpy
from rclpy.node import Node

class MinimalPublisher(Node):

    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = 'ہیلو ورلڈ: %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('شائع کر رہا ہے: "%s"' % msg.data)
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### سبسکرائب نوڈ
```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalSubscriber(Node):

    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info('میں نے سنا: "%s"' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    minimal_subscriber = MinimalSubscriber()
    rclpy.spin(minimal_subscriber)
    minimal_subscriber.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## عام مسائل
- **نوڈ دریافت کے مسائل**: نیٹ ورک کنفیگریشن یا DDS ایمپلیمنٹیشن تنازعات کی وجہ سے نوڈس رابطہ نا کام ہو سکتے ہیں
- **QoS عدم مطابقت**: غیر مطابقت رکھنے والے QoS کے سیٹنگز کے ساتھ پبلشرز اور سبسکرائبرز رابطہ نہیں کر سکتے
- **ریسورس مینجمنٹ**: نوڈس کو مناسب طریقے سے تباہ کرنا نہیں بھولنا میموری لیکس کا سبب بن سکتا ہے
- **تھریڈنگ کے مسائل**: کال بیکس کو غلط طریقے سے ہینڈل کرنا ریس کنڈیشنز کا سبب بن سکتا ہے
- **پیکج ڈیپینڈنسیز**: غائب یا غلط پیکج ڈیپینڈنسیز نوڈس کو چلنے سے روک سکتی ہیں

## چیک پوائنٹس / مینی ایکسائزز
1. ایک پبلشر نوڈ تیار کریں جو ٹیمپریچر ریڈنگز شائع کرے اور ایک سبسکرائبر جو انہیں کنسول میں لاگ کرے
2. ایک سروس نافذ کریں جو دو پوائنٹس کے درمیان فاصلہ معلوم کرے اور اسے ایک کلائنٹ کے ساتھ ٹیسٹ کریں
3. ایک لانچ فائل سیٹ اپ کریں جو مختلف QoS سیٹنگز کے ساتھ متعدد نوڈس شروع کرے
4. ایک کسٹم میسج ٹائپ بنائیں اور اسے ایک پبلشر-سبسکرائب جوڑی میں استعمال کریں
5. ROS 2 کمانڈ لائن ٹولز کا استعمال کرتے ہوئے نوڈس کے درمیان کمیونیکیشن کا مسئلہ ڈیبگ کریں
